# Запуск разработки OpenWeatherAPI

ТЗ проекта находится в файле `TZ.md`. 

**Задача:** Сгенерируй полный production-ready код проекта строго по техническому заданию.

## Порядок генерации файлов:

1. `requirements.txt`
2. `.env.example`
3. `exceptions.py`
4. `config.py`
5. `utils.py`
6. `client.py` ← основной модуль
7. `cli.py`
8. `README.md`

## Критические правила:

- Используй **ТОЛЬКО** endpoint'ы, указанные в разделе 2.1 ТЗ
- Строго соблюдай структуру JSON из примеров в ТЗ
- Весь пользовательский текст — на **русском** языке
- Никаких `print()`, только `logger`
- Никаких `pass` / `TODO` / заглушек
- Type hints и docstrings обязательны
- Код должен запускаться через `python cli.py current "Москва" --extended`

## Перед генерацией:

1. Прочитай `TZ.md` целиком
2. Подтверди готовность и сгенерируй **первый файл** (`requirements.txt`)
3. Жди моего сигнала перед переходом к следующему файлу

## Формат вывода каждого файла:

```python
# === filename.py ===
# Краткое описание модуля
# =======================

<код файла>
```

Начинай с подтверждения, что ТЗ прочитано, и приступай к `requirements.txt`.
